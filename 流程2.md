## MVC 架构规则
Model 层纯 Python 实现
所有数据操作（文件解析、SQLite管理、聚类参数配置）均不依赖 PySide6，可独立测试，仅通过返回数据对象或信号与外部交互。

View 层无业务逻辑
负责 UI 构建、用户交互与数据展示，不包含数据处理、文件读写等逻辑。View 通过信号发射用户操作，通过槽接收更新后的数据。

Controller 层（Worker）为计算核心
负责将 View 的配置请求转化为计算任务，生成结果后通过信号回传。Controller 不直接操作 UI，也不依赖具体数据存储方式。

层间通信明确分离

View ↔ Controller：通过 Signal/Slot 传递配置参数和结果数据

Controller → Model：调用方法获取数据，返回计算结果（Model 无 Qt 依赖）

数据载体：采用 dataclass（如 ClusteringParams、ClusteringResult）规范化传输，避免裸 dict/DataFrame 跨层耦合

## 2. 数据导入与智能解析 (models.py 中的 DataManager 类)
### 2.1 文件加载
- 读取 Excel 时，先用 get_excel_sheets() 获取所有 sheet 名。
- 若仅 1 个 sheet：直接加载。
- 若多个 sheet：弹出 QInputDialog 下拉列表让用户选择，同时提供“合并所有 sheet”选项（合并时自动增加 source_sheet 列记录来源）。
- 兼容 CSV 文件直接加载，无需选 sheet。

### 2.2 表头自动定位（核心优化）
- 遍历所有行，寻找“表头候选行”：优先搜索第一个包含"物体编号"的单元格（支持字符串包含匹配）。
- 若未找到"物体编号"，对前 20 行计算综合评分：评分 = 映射关键词命中数 × 非空单元格比例。每个命中词 +1 分（映射关键词包括："粒宽""粒高""高(mm)""周长""面积""相似度"），选择综合评分最高的行作为表头。
- 将该行设为列名，该行以上的所有行（元数据）直接丢弃；该行以下作为数据区。
- 注意：某些 Excel 中可能包含多个数据块（即下方再次出现"物体编号"行），需向下扫描检测，若发现第二个数据块，自动拆分并分别处理，同时提取上方"品种名称"作为分组标签，添加 source_group 列。

### 2.3 列名标准化映射
- 建立双向映射字典（utils.py 中定义）：
  正向：{"粒宽":"width", "粒高":"height", "高(mm)":"height", "高":"height", "周长":"perimeter", "面积(mm^2)":"area", "面积":"area", "相似度":"similarity", "物体编号":"object_id"}
  反向：用于导出中文表头（width→粒宽, height→粒高, perimeter→周长, area→面积, similarity→相似度）
- 使用正则忽略列名中的单位后缀和括号，优先匹配核心关键词。
- 未能匹配的列标记为“非特征列”，不在特征勾选框显示，但保留在表格中。

### 2.4 数据清洗
- 将标准化后的数值列强制转为 float，转换失败置为 NaN。
- 过滤无效行：数据区中至少包含 2 个非空数值单元格的行才保留（排除全空行和统计摘要行，如"表型数据条数:6"）。
- 统计有效行数，若 < 5 则弹出 QMessageBox.warning 提示"有效数据量不足"并停止加载。

### 2.5 内存数据库存储
- 将清洗后的 DataFrame 写入 sqlite3 内存数据库（表名：phenotype_data），列名使用英文映射名。
- 增加固定列：row_id（自增主键）、source_sheet、source_group、cluster_id（初始为 NULL）。
- 使用 QSqlTableModel 驱动 QTableView，通过自定义 headerData 方法将英文列名映射为中文显示（仅对表头列名生效，数据库内部保持英文）。
- 导入完成后显示状态栏信息：识别到的数据块数量、每个块有效行数、丢弃行数。

## 3. 参数配置（左侧面板）
### 3.1 特征选择
- 动态生成 QCheckBox：排除 object_id、图片路径、ID、备注、坐标等非数值元数据列，仅展示已识别的特征列（width, height, perimeter, area, similarity）。
- 最少勾选 2 个特征才能启用“运行聚类”按钮。

### 3.2 K 值设置
- QSpinBox（最小 2，最大 20），默认 3。
- QRadioButton “自动优化K值”：选中后 SpinBox 禁用，聚类时传入 n_clusters=None。

## 4. 多线程聚类 (workers.py 中的 ClusteringWorker)
### 4.1 线程模式
- 使用 QObject + moveToThread 模式（不继承 QThread），通过 @Slot 装饰的方法启动计算。
- 定义信号：progress_signal(int)、finished_signal(dict)、error_signal(str)。

### 4.2 计算流程
- 从数据库读取用户选中的特征列数据，构造 DataFrame。
- 调用 run_phenotype_clustering(df, features=selected_features, n_clusters=k_or_None)。
  * 要求修改 analyzer.py 函数签名：增加 features: list 参数，内部使用传入的 features 列进行聚类，而非硬编码。
  * 如果无法修改原函数，在 worker 中用 monkey-patch 临时替换其内部特征列表。
- 计算时主窗口显示 QProgressDialog（范围 0, 0 的不确定进度条），点击取消则调用 worker.stop() 终止。
- 计算完成后，将聚类结果（df_with_cluster_id, centroids_df）通过信号传出。

### 4.3 结果写回
- 在 phenotype_data 表增加 cluster_id 列（若不存在）。
- 使用 df_with_cluster_id 的原始索引精确更新对应行的 cluster_id；被算法丢弃的行（含 NaN/负值）保持 NULL。
- 更新 QSqlTableModel 并刷新 QTableView。

## 7. 数据流与架构约束
- 定义数据类（dataclass）在层间传递：ClusteringParams（特征列表、K值、自动优化标志）、ClusteringResult（聚类后 DataFrame、质心 DataFrame）。
- Model 层只负责数据存取与状态管理。
- View 层只负责 UI 创建与信号转发，不直接操作数据。
- Worker 层只负责计算，通过信号与主线程通信。
- 不使用全局变量，配置通过信号/槽或依赖注入传递。

### 8. 多品种聚类策略（重要）
#### 8.1 基本原则
- 聚类分析默认**按品种（source_group）分组独立进行**，每个品种单独计算 cluster_id。
- 品种标识来源于数据块的“品种名称”列（见2.2），若缺失则视为单一品种。

#### 8.2 UI 控制
- 左侧面板顶部新增 QLabel 显示当前品种数量。
- 可视化区域顶部新增 QComboBox“选择品种”，切换后表格和图表同步刷新为对应品种的聚类结果。
- 参数配置中的“特征选择”和“K值”对所有品种共用（保持分析标准一致）。

#### 8.3 多品种聚类执行
- ClusteringWorker 接收品种列表，循环调用 run_phenotype_clustering，为每个品种生成独立 cluster_id（从0开始）。
- 最终返回 dict 结构：{品种名: (df_with_cluster_id, centroids_df)}，写入数据库时 cluster_id 加上品种前缀避免冲突（如 "rice_0"）。

#### 8.4 导出增强
- 导出时增加“品种名称”列，cluster_id 显示为数字（0,1,2...），同一品种内连续。
- 若用户选择合并品种导出，自动拆分 sheet，每个品种一个 sheet。
