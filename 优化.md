9. 多品种识别与选择功能（新增章节）
9.1 品种名称自动提取
9.1.1 提取规则（优先级从高到低）
text
1. 元数据区“品种名称”列的值（第2行D列，最可靠）
2. 多数据块场景下，每个数据块上方元数据区独立提取
3. 若元数据中未找到“品种名称”，降级使用 sheet 名称
4. 最终兜底：提示用户手动输入品种名
9.1.2 具体提取逻辑
定位到第一个“物体编号”表头行后，向上回溯前20行元数据区

搜索包含“品种名称”的单元格，提取该列对应数据行的值

示例：第1行D列标题为“品种名称”，第2行D列值为“水稻”→ 提取“水稻”

若同一sheet内检测到第二个“物体编号”行（多数据块），分别提取每个数据块上方的品种名称

示例：多品种汇总_单sheet.xlsx 中，第一个数据块品种=“油菜”，第二个=“水稻”

9.1.3 品种名称标准化
去除首尾空格、换行符

统一命名规范：如“水稻”、“油菜”、“小麦”等

若品种名称为空或纯数字，标记为 "未知品种_N"（N为序号）

python

切换操作	表格更新	图表更新	聚类按钮状态

9.3 聚类执行模式
9.3.1 单品种聚类（默认强制）
当前选中品种 → 仅对该品种数据执行聚类

聚类结果仅更新该品种行的 cluster_id

切换品种后，表格显示对应品种的聚类结果

切换到“全部品种”时，cluster_id 列显示 "品种名_簇号" 格式

9.3.2 全品种聚类（可选，需用户确认）
添加复选框 [☐ 合并所有品种聚类（不推荐）]

勾选时弹窗警告：

text
⚠️ 警告：合并不同品种进行聚类可能导致生物学意义丧失。
品种间天然差异（如油菜面积远大于水稻）会主导聚类结果，
品种内的表型分化将被掩盖。建议按品种分别聚类分析。

[ 取消 ] [ 仍然合并 ]
用户确认后，ClusteringWorker 传入所有品种数据进行合并聚类

9.4 数据库结构调整
phenotype_data 表增加 variety_name 列（TEXT，从元数据提取）

source_sheet 保留用于记录数据来源

cluster_id 在单品种模式下为整数（0,1,2...），全品种模式下为 "品种名_簇号" 格式

9.5 可视化品种区分
选中“全部品种”时，散点图按 variety_name 着色，聚类结果不可用

选中单一品种时，散点图按 cluster_id 着色，显示聚类中心

图表标题动态显示："水稻 - PCA聚类结果 (K=3)"
